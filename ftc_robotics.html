About FTC Robotics

As the lead developer for my high school's FTC Robotics team, I helped design and build a competition-ready robot. It was a six-person team - I handled software and electronics. I programmed the robot using Java and developed communication logic between two Android phones: one mounted on the robot, and the other attached to an Xbox controller for remote driving. I also wired and soldered all electrical components. Our robot successfully competed in statewide tournaments, where we placed first in our division and earned medals. Although we didn't advance to nationals, the project served as a key foundation for my hands-on experaince with embeded systems, teamwork under pressure, and real-time control systems.

Technologies

The robot was programmed in Java using the FTC SDK, running on two Android devices connected via Wi-Fi Direct. The control system used an Xbox gamepad plugged into the driver-side phone, which communicated commands wirelessly to the phone mounted on the robot. This phone interfaced with motors, sensors, and servos. I handled the full wiring and soldering of all hardware components, including power distribution, signal routing, and motor controller connections. The project involved tuning PID loops, troubleshooting sensor noise, and optimizing motor control for autonomous and driver-controlled modes.

Java C++ FTC

Development Process

The development process began with designing the control system architecture and selecting hardware components compatible with the FTC ecosystem. I wrote Java code to control movement, arm positioning, and autonomous routines, while manually wiring and soldering the entire electronic system. I iterated through multiple testing cycles to resolve timing issues and control lag, and worked closely with the mechanical team to align software with physical limitations. Collaboration and debugging were key, especially during competitions where rapid problem-solving was critical.

Usage

During matches, the robot was operated by connecting a game controller to the driver-side Android phone, which sent input commands to the robot's onboard phone. The robot executed tasks like object pickup, navigation, and scoring under both driver and autonomous control. As lead programmer and hardware integrator, I ensured smooth communication, responsive control, and minimal downtime - all of which helped us succeed in the competition environment.
